mdBook

A guide on how to set up a website using mdBook, a documentation tool built with Rust.

Setup

First ensure you have the latest Rust version installed, then install mdBook via Cargo. After initialization with mdbook init, you'll have a project structure with a src directory for markdown files and a book directory for rendered output.

Development Workflow

For active development, there are three rendering options:

  • Building: Run mdbook build to generate static output
  • Watching: Use mdbook watch for automatic rebuilds on file changes
  • Serving: Execute mdbook serve to preview changes at localhost:3000 with hot reloading

Content Organization

The SUMMARY.md file structures your book using three chapter types: prefix chapters (unnumbered introductory content), numbered chapters (main content with optional nesting), and suffix chapters (concluding unnumbered sections).

Deployment

To host on GitHub Pages, create a gh-pages branch and populate its root directory with the mdbook build output.
